### 问题描述
`KingScript` 中 `Date` 使用的是 `Java` 的 Calendar 封装的，所以在使用上与原生 JavaScript会有所区别

### 常见问题如下

1）`KingScript` 在使用 `Date` 时无法通过运算符 `>` `<` `=` 进行比较

#### 问题分析
`KingScript` 中 `Date` 是对 `Java` 的 `Date` 对象进行了封装，`Java` 中 `Date` 对象无法直接通过运算符比较。
```kingscript
let date1:Date = new Date("2024-12-01");
let date2:Date = new Date("2025-01-01");
console.log(date1 <= date2);
//false
```

#### 解决办法
Date对象的比较，通过 `compareTo` 方法或者 `getTime` 后进行比较：
```kingscript
let date1:Date = new Date("2024-12-01");
let date2:Date = new Date("2025-01-01");
console.log(date1.compareTo(date2) < 0);
//true
console.log(date1.getTime() < date2.getTime());
//true
```

2）`KingScript` 中 `Date` 的 `getDay()` 方法返回值是 `1` ~ `7`，而不是 `0` ~ `6`

#### 原因说明
```
1. JavaScript 的 Date.getDay()
   返回值范围：0 ~ 6
   起始逻辑：
   0 = Sunday, 1 = Monday, ..., 6 = Saturday
   （从 Sunday 开始）
   javascript
   const date = new Date();
   console.log(date.getDay()); // 示例：假设今天是周四，输出 4
```
```
2. KingScript 使用的是 Java 的 Calendar.get(Calendar.DAY_OF_WEEK)
   返回值范围：1 ~ 7
   起始逻辑：
   1 = Sunday, 2 = Monday, ..., 7 = Saturday
   （同样从 Sunday 开始）
   java
   Calendar calendar = Calendar.getInstance();
   System.out.println(calendar.get(Calendar.DAY_OF_WEEK));
   // 示例：假设今天是周四，输出 5
```

#### 关键区别总结
| 语言                 | 方法/字段                | 范围    | 星期日 | 星期四 |
|--------------------|----------------------|-------|-----|-----|
| JavaScript         | Date.getDay()        | 0 ~ 6 | 0   | 4   |
| KingScript / Java  | Calendar.DAY_OF_WEEK | 1 ~ 7 | 1   | 5   |
