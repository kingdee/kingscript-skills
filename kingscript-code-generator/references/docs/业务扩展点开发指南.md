# 业务扩展点开发指南

## 概述

业务扩展点（Extension Point）是金蝶云苍穹提供的二次开发机制，用于在不修改标准产品源码的前提下，通过注册自定义逻辑来干预或增强标准业务流程。

与插件（Plugin）不同，扩展点：
- **不是 class**，通过**对象字面量**实现接口
- 通常只包含 **1-3 个方法**
- 没有生命周期概念，只在特定业务时机触发
- 导入的是**业务领域接口**，不是通用插件接口

## 基本结构

```ts
// TODO: 根据实际业务场景替换为对应的扩展点接口
import { IYourExtensionPoint } from '@constellation/xxx/extpoint';

let ext: IYourExtensionPoint = {
  // TODO: 替换为实际接口定义的方法名和签名
  yourMethod(args: any): boolean {
    // 实现业务逻辑
  }
};

export { ext };
```

**要点**：
- 用 `let` 声明对象字面量，类型标注为扩展点接口
- 导出必须是 `export { ext }`（不是 `export default`，也不是 `export { plugin }` 或 `export { kwcController }`）
- 接口路径需根据业务领域替换，格式如： `@constellation/xxx/extpoint`、`@cosmic/xxx/extpoint`、`@galaxy/xxx/extpoint`

## 约束

扩展点遵循 `gotchas/` 中的通用保守约束（详见 `../../gotchas/README.md` 坑位速查），包括但不限于：禁用 `?.`/`??`、金额保守转换、禁用 `static`、异常用 `'' + e`、`for-of` 禁止、`entryentity.` 前缀、`row.getDate()` 禁止等。

> 安全读取字段、遍历分录、异常处理的完整代码写法，请参考 `../../templates/extpoint-template.md` 和 `../../gotchas/` 对应专题。
