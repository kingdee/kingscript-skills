# 插件开发指南

## 概述

插件（Plugin）是金蝶云苍穹最主要的二次开发机制，通过继承平台基类并重写生命周期方法，实现对页面交互、单据业务、列表过滤、操作校验等场景的定制化开发。

插件的核心特征：
- 基于 **Class 继承**，重写基类提供的生命周期方法
- 固定导出方式：`let plugin = new MyPlugin(); export { plugin };`
- 触发时机由**平台事件驱动**（如页面加载、字段变更、操作提交等）
- 基类和事件参数均从 `@cosmic/bos-core/...` 导入

## 基本结构

```ts
import { AbstractFormPlugin } from "@cosmic/bos-core/kd/bos/form/plugin";
import { AfterBindDataEvent } from "@cosmic/bos-core/kd/bos/form/events";

class MyPlugin extends AbstractFormPlugin {

  afterBindData(e: AfterBindDataEvent): void {
    super.afterBindData(e);
    // 在数据绑定后执行业务逻辑
  }

}

let plugin = new MyPlugin();

export { plugin };
```

## 约束

插件遵循 `gotchas/` 中的通用保守约束（详见 `../../gotchas/README.md` 坑位速查），包括但不限于：禁用 `?.`/`??`、金额保守转换、禁用 `static`、异常用 `'' + e`、`for-of` 禁止、`row.getDate()` 禁止等。

**插件专属约束**：

- 导出必须是 `export { plugin }`（不是 `export default`，也不是 `export { ext }` 或 `export { kwcController }`）

## 插件类型速查

| 插件类型 | 基类 | 导入路径 | 典型场景 |
|---------|------|---------|---------|
| 表单插件 | `AbstractFormPlugin` | `@cosmic/bos-core/kd/bos/form/plugin` | 普通页面生命周期、字段回传、关闭回调 |
| 单据插件 | `AbstractBillPlugIn` | `@cosmic/bos-core/kd/bos/bill` | 单据字段联动、分录增删改、F7、操作校验 |
| 列表插件 | `AbstractListPlugin` | `@cosmic/bos-core/kd/bos/list/plugin` | 列表过滤、选择联动、列表回调 |
| 操作插件 | `AbstractOperationServicePlugIn` | `@cosmic/bos-core/kd/bos/entity/plugin` | 服务端操作校验、准备属性、校验器 |
| 打印插件 | `AbstractPrintPlugin` | `@cosmic/bos-core/kd/bos/print/core/plugin` | 打印前后处理、打印数据包扩展 |
| 报表表单插件 | `AbstractReportFormPlugin` | `@cosmic/bos-core/kd/bos/report/plugin` | 报表过滤面板、默认查询参数 |
| 报表查询插件 | `AbstractReportListDataPlugin` | `@cosmic/bos-core/kd/bos/entity/report` | 报表数据查询、列定义 |
| 转换插件 | `AbstractConvertPlugIn` | `@cosmic/bos-core/kd/bos/entity/botp/plugin` | 单据转换下推、字段映射 |
| 引入插件 | `BatchImportPlugin` | `@cosmic/bos-core/kd/bos/form/plugin/impt` | 批量引入、字段映射、界面锁定 |
| 引出插件 | `AbstractListPlugin` | `@cosmic/bos-core/kd/bos/list/plugin` | 导出前筛选、格式控制 |
| 后台任务插件 | `AbstractTask` | `@cosmic/bos-core/kd/bos/schedule` | 调度任务、长耗时异步处理 |
| 移动表单插件 | `AbstractMobFormPlugin` | `@cosmic/bos-core/kd/bos/form/plugin` | 移动端表单、定位、刷新、上传 |
| 工作流插件 | `WorkflowPlugin` | `@cosmic/bos-core/kd/bos/workflow/engine/extitf` | 参与人计算、条件判断、通知 |

## 高频事件速查

### 表单/单据插件

| 事件 | 说明 | 常用场景 |
|------|------|---------|
| `afterBindData` | 数据绑定后 | 初始化界面状态 |
| `beforeBindData` | 数据绑定前 | 动态修改绑定数据 |
| `propertyChanged` | 字段值变更后 | 字段联动 |
| `beforePropertyChanged` | 字段值变更前 | 拦截字段变更 |
| `beforeFieldPostBack` | 字段回传前 | 控制 F7 回传行为 |
| `beforeDoOperation` | 操作执行前 | 保存/提交前校验 |
| `afterDoOperation` | 操作执行后 | 操作后处理 |
| `closedCallBack` | 关闭回调 | 子页面返回后刷新 |
| `beforeClosed` | 页面关闭前 | 关闭确认、数据保存提示 |
| `registerListener` | 注册监听器 | 自定义控件事件监听 |

### 列表插件

| 事件 | 说明 | 常用场景 |
|------|------|---------|
| `listBeforeBindData` | 列表数据绑定前 | 默认过滤条件 |
| `dataSelect` | 选中行变更 | 按钮状态联动 |
| `billClosedCallBack` | 单据关闭回调 | 列表刷新 |
| `beforePackageData` | 数据打包前 | 列表输出扩展 |

### 操作插件

| 事件 | 说明 | 常用场景 |
|------|------|---------|
| `onPreparePropertys` | 准备属性 | 补齐字段减少后续查询 |
| `onAddValidators` | 添加校验器 | 前置校验 |
| `onBeginOperationTransaction` | 操作事务开始 | 事务内业务处理 |
| `onEndOperationTransaction` | 操作事务结束 | 事务后处理 |

## 选型决策

```
需要干预什么？
├── 页面交互 → 表单插件 or 单据插件
│   ├── 普通页面/看板/查询 → AbstractFormPlugin
│   └── 单据页面（有分录、F7、操作） → AbstractBillPlugIn
├── 列表 → 列表插件（AbstractListPlugin）
├── 服务端操作校验 → 操作插件（AbstractOperationServicePlugIn）
├── 打印 → 打印插件（AbstractPrintPlugin）
├── 报表
│   ├── 过滤面板/表单 → 报表表单插件（AbstractReportFormPlugin）
│   └── 数据查询/列定义 → 报表查询插件（AbstractReportListDataPlugin）
├── 单据转换/下推 → 转换插件（AbstractConvertPlugIn）
├── 引入/导出
│   ├── 批量引入 → 引入插件（BatchImportPlugin）
│   └── 导出扩展 → 引出插件（AbstractListPlugin）
├── 后台任务/调度 → 后台任务插件（AbstractTask）
├── 移动端 → 移动表单插件（AbstractMobFormPlugin）
└── 审批流/工作流 → 工作流插件（WorkflowPlugin）
```

## 起手建议

1. **先选对基类**：表单 vs 单据是最常见的混淆点。有分录、F7 的场景用 `AbstractBillPlugIn`，普通页面用 `AbstractFormPlugin`。
2. **少量方法起步**：不要一上来就补满所有生命周期方法，只保留当前需求真的会用到的 1-3 个方法。
3. **先看示例再写**：`../../examples/plugins/插件示例/` 下有按类型和场景拆分的示例，先找最接近的再写。
4. **补齐 import**：新增事件参数、助手类、枚举时，必须同步补齐 import，不依赖 IDE 自动导入。
5. **对照 SDK 确认**：写完方法签名后，回跳到 `../../sdk/packages/` 下对应事件包确认参数语义。

## 输出前自检

- 是否选对了基类？（表单 vs 单据 vs 列表）
- 导出是否为 `export { plugin }`？（不是 `export default`，也不是 `export { ext }` 或 `export { kwcController }`）
- 每个新增的 import 是否已显式声明？
- 通用运行时约束自检请对照 `../../gotchas/README.md` 坑位速查
